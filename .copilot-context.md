# Project Context for GitHub Copilot

## C# Version: 5.0 or earlier (C# 4.0/5.0)

This project uses an **older C# version** with limited language features.

### ❌ NOT SUPPORTED - Do NOT use these features:

1. **String interpolation** - `$"Hello {name}"` ❌
   - ✅ Use: `"Hello " + name`

2. **Target-typed new expressions** - `new()` ❌
   - ✅ Use: `new List<string>()`

3. **Auto-property initializers** - `public int Value { get; set; } = 10;` ❌
   - ✅ Use: Private backing field with default value

4. **Null-conditional operator** - `obj?.Method()` ❌
   - ✅ Use: `if (obj != null) { obj.Method(); }`

5. **Null-coalescing assignment** - `x ??= value` ❌
   - ✅ Use: `if (x == null) { x = value; }`

6. **Expression-bodied members** - `public int Value => _value;` ❌
   - ✅ Use: Full property with `get { return _value; }`

7. **Nullable reference types** - `string?` ❌
   - ✅ Use: `string` (no `?` operator)

8. **Null-forgiving operator** - `obj!.Method()` ❌
   - ✅ Use: `obj.Method()`

9. **Pattern matching** - `if (obj is string s)` ❌
   - ✅ Use: Traditional type checks and casts

10. **Tuple syntax** - `(int, string)` ❌
    - ✅ Use: Classes or out parameters

11. **Discard operator** - `_ = value;` ❌
    - ✅ Use: Just call the method without assignment, or use a dummy variable

12. **Exception filters** - `catch (Exception ex) when (condition)` ❌
    - ✅ Use: Traditional catch with if-condition inside the block

### ✅ SUPPORTED - Safe to use:

- Traditional properties with explicit `get`/`set` blocks
- `var` keyword for local variables
- LINQ (Language Integrated Query)
- `async`/`await` (introduced in C# 5.0)
- Extension methods
- Lambda expressions
- Anonymous types
- Automatic properties (without initializers): `{ get; set; }`
- String concatenation with `+`
- Traditional null checks: `if (obj != null)`

## Project Structure

- **Entry Point**: `rotation.cs` - Contains the main rotation class
- **Initialization**: Use `LoadSettings()` method for startup code
- **Framework**: Inferno WoW bot framework
  - Static methods like `Inferno.PrintMessage()`, `Inferno.Cast()`, etc.
  - Uses string concatenation (no interpolation support)

## Build Notes

- This is a library project (loaded by Inferno framework)
- Missing `Main` entry point is expected
- Build with: `dotnet build -p:OutputType=Library`

## Build Script Requirements ⚠️

**IMPORTANT**: This project uses a build script (`Build.ps1`) that merges files together.

### When Adding New `using` Statements:

**You MUST update TWO files:**

1. **Add to `Scripts\AddToStart.ps1`** - So they appear at the top of merged file
2. **Add to `Scripts\LinesToExclude.ps1`** - So they get stripped from individual files during merge

Example:
```powershell
# Scripts/AddToStart.ps1
$linesToAddAtStart = @(
    "using System;"
    "using System.Collections.Generic;"
    "using System.Net;"
    # ... add new using here
)

# Scripts/LinesToExclude.ps1
$linesToExclude = @(
    "using System;"
    "using System.Collections.Generic;"
    "using System.Net;"
    # ... add new using here
)
```

### Currently Configured Using Statements:
- `using System;`
- `using System.Collections.Generic;`
- `using System.Net;`
- `using System.Net.WebSockets;`
- `using System.Text;`
- `using System.Threading;`
- `using System.Threading.Tasks;`
- `using System.Drawing;`
- `using InfernoWow.API;`

### Build Process Flow:

```
Source Files (rotation.cs, WebSocket/*.cs, etc.)
    ↓
Build.ps1 executes BuildFile.ps1:
  1. Strips using statements from file bodies (LinesToExclude.ps1)
  2. Adds all using statements at top once (AddToStart.ps1)
  3. Merges all files together
  4. Removes duplicate newlines
    ↓
Output: C:\libs\Live\Rotations\Retail\PenelosGambits\rotation.cs
```

### ALWAYS Execute Build.ps1 After Changes:

```powershell
cd C:\Repos\PenelosGambitsInferno2.0\PenelosGambits
.\Build.ps1
```

**The merged output file is what runs in production, NOT the source files!**

### Source File Guidelines:

- ✅ Keep using statements in source files (for IDE IntelliSense support)
- ✅ Build script will automatically strip and re-add them
- ✅ Write normal C# code in source files
- ✅ Build script handles the merging
- ⚠️ Never manually edit the output file at `C:\libs\Live\Rotations\...`

## When Writing Code

**Always use legacy C# syntax:**
- Explicit type names in constructors
- Traditional null checks with `if`
- String concatenation with `+`
- Full property syntax with backing fields for initialization
- No modern nullable annotations

**After every code change:**
1. Save your files
2. Run `.\Build.ps1` to merge and deploy
3. The merged file is what runs in production

**When adding new using statements:**
1. Add to both `Scripts\AddToStart.ps1` AND `Scripts\LinesToExclude.ps1`
2. Keep the using in your source file for IDE support
3. Run `.\Build.ps1` to apply changes

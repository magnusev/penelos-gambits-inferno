# Project Context for GitHub Copilot

## C# Version: 5.0 or earlier (C# 4.0/5.0)

This project uses an **older C# version** with limited language features.

### ❌ NOT SUPPORTED - Do NOT use these features:

1. **String interpolation** - `$"Hello {name}"` ❌
   - ✅ Use: `"Hello " + name`

2. **Target-typed new expressions** - `new()` ❌
   - ✅ Use: `new List<string>()`

3. **Auto-property initializers** - `public int Value { get; set; } = 10;` ❌
   - ✅ Use: Private backing field with default value

4. **Null-conditional operator** - `obj?.Method()` ❌
   - ✅ Use: `if (obj != null) { obj.Method(); }`

5. **Null-coalescing assignment** - `x ??= value` ❌
   - ✅ Use: `if (x == null) { x = value; }`

6. **Expression-bodied members** - `public int Value => _value;` ❌
   - ✅ Use: Full property with `get { return _value; }`

7. **Nullable reference types** - `string?` ❌
   - ✅ Use: `string` (no `?` operator)

8. **Null-forgiving operator** - `obj!.Method()` ❌
   - ✅ Use: `obj.Method()`

9. **Pattern matching** - `if (obj is string s)` ❌
   - ✅ Use: Traditional type checks and casts

10. **Tuple syntax** - `(int, string)` ❌
    - ✅ Use: Classes or out parameters

### ✅ SUPPORTED - Safe to use:

- Traditional properties with explicit `get`/`set` blocks
- `var` keyword for local variables
- LINQ (Language Integrated Query)
- `async`/`await` (introduced in C# 5.0)
- Extension methods
- Lambda expressions
- Anonymous types
- Automatic properties (without initializers): `{ get; set; }`
- String concatenation with `+`
- Traditional null checks: `if (obj != null)`

## Project Structure

- **Entry Point**: `rotation.cs` - Contains the main rotation class
- **Initialization**: Use `LoadSettings()` method for startup code
- **Framework**: Inferno WoW bot framework
  - Static methods like `Inferno.PrintMessage()`, `Inferno.Cast()`, etc.
  - Uses string concatenation (no interpolation support)

## Build Notes

- This is a library project (loaded by Inferno framework)
- Missing `Main` entry point is expected
- Build with: `dotnet build -p:OutputType=Library`

## When Writing Code

**Always use legacy C# syntax:**
- Explicit type names in constructors
- Traditional null checks with `if`
- String concatenation with `+`
- Full property syntax with backing fields for initialization
- No modern nullable annotations
